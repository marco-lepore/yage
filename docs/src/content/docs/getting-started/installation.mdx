## Install

Every YAGE module is published as its own npm package under the `@yage/*`
scope. Install only what you need:

```bash
npm install @yage/core @yage/renderer
```

The full list of packages:

| Package | Purpose |
|---|---|
| `@yage/core` | ECS, math, events, scheduling |
| `@yage/renderer` | PixiJS v8 rendering pipeline |
| `@yage/physics` | Rapier 2D physics integration |
| `@yage/input` | Keyboard, mouse, and gamepad |
| `@yage/audio` | Sound playback and mixing |
| `@yage/particles` | Particle emitters and effects |
| `@yage/tilemap` | Tile-based level support |
| `@yage/ui` | In-game UI primitives |
| `@yage/ui-react` | React bindings for game UI |
| `@yage/debug` | Dev tools and inspectors |
| `@yage/save` | Save / load state management |

`@yage/core` sits at the base of the dependency graph — every other package
depends on it, so it is always installed automatically.

## Prerequisites

- **Node.js** 18 or later
- **npm** 9+ (or any compatible package manager — pnpm and yarn work fine)
- A bundler that supports TypeScript and ESM (Vite is recommended)

## Vite Setup

If you use `@yage/physics`, its Rapier 2D engine requires WebAssembly. Vite
needs the `vite-plugin-wasm` plugin to handle `.wasm` imports:

```bash
npm install -D vite-plugin-wasm
```

```typescript title="vite.config.ts"
import { defineConfig } from "vite";
import wasm from "vite-plugin-wasm";

export default defineConfig({
  plugins: [wasm()],
});
```

## Verifying the Install

Create a minimal entry file and run your dev server to confirm everything is
working:

```typescript
import { Engine } from "@yage/core";
import { RendererPlugin } from "@yage/renderer";

const engine = new Engine();
engine.use(new RendererPlugin({ width: 800, height: 600, backgroundColor: 0x1a1a2e }));
await engine.start();
```

If a blank game window appears with the configured background colour, YAGE is
installed correctly.
